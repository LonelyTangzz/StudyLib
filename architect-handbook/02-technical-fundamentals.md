# 02 · 技术基础

> 架构师做的所有"高层决策"，根都扎在五个基础学科里：**编程语言、数据结构与算法、操作系统、计算机网络、数据库**。任何一个根有缺，决策就会浮在空中。

这一章不是教你从零学这五门课，而是告诉你**架构师必须掌握到什么程度、为什么**。

---

## 1. 编程语言

### 1.1 至少精通一门、了解两类范式

**精通一门**：至少能讲清楚——
- 内存模型（栈/堆怎么分配、对象怎么布局、GC 怎么工作 / 没有 GC 怎么管理）
- 并发模型（线程？协程？Actor？事件循环？）
- 编译/解释/JIT 链路
- 标准库的关键抽象（集合、IO、网络、并发）

**了解两类范式**：建议至少触过一种**静态类型 + 系统级**（Go / Rust / Java / C++）和一种**动态类型 + 高表达力**（Python / JavaScript / Ruby）。两类语言写久了，你会理解很多取舍是**语言层面就已经定死的**。

### 1.2 架构师视角的语言取舍

| 维度 | 静态类型语言强 | 动态类型语言强 |
|---|---|---|
| 大规模重构安全性 | ✓ | |
| 跑通到上线的速度 | | ✓ |
| 编译期问题发现 | ✓ | |
| 团队新人上手 | | ✓ |
| 长期维护 | ✓ | |
| 短项目/原型 | | ✓ |

**架构含义**：选语言不只看"我们组用什么"，要看团队规模、生命周期、性能要求。一个 5 人小队做 6 个月的项目，可能 Python/Node 比 Java 总成本更低。

### 1.3 常被忽略但必须懂的语言级概念

- **错误处理范式**：异常 vs 返回值（Go 的 err、Rust 的 Result）。决定接口设计风格。
- **泛型/类型推导**：影响 API 抽象能力。
- **不可变性**：函数式语言强制，命令式语言可选。直接影响并发安全。
- **接口/协议/Trait**：决定怎么做依赖反转。
- **包管理与依赖图**：npm 黑洞、Maven 树深、Go modules、Cargo —— 每种都有它的坑。

## 2. 数据结构与算法

### 2.1 必须熟练（不是面试熟练，是脑子里有的)

| 数据结构 | 你必须知道 |
|---|---|
| 数组 / 动态数组 | 连续内存、扩容代价、缓存友好 |
| 链表 | 不缓存友好、随机访问慢 |
| 哈希表 | 冲突解决、负载因子、扩容、最坏退化到 O(n) |
| 树：BST / AVL / 红黑 | 平衡的代价、为什么数据库不用它们做磁盘索引 |
| **B / B+ 树** | 数据库索引的基础，扇出大、磁盘友好 |
| 堆 | 优先队列、定时任务 |
| Trie | 前缀匹配、路由、词典 |
| 跳表 (Skip List) | Redis Sorted Set 的实现 |
| 布隆过滤器 (Bloom Filter) | 缓存穿透、去重 |
| LSM Tree | RocksDB / LevelDB / Cassandra 写优化的基础 |
| 一致性哈希 (Consistent Hashing) | 分片、负载均衡 |
| HyperLogLog | 基数估算（UV 统计） |

**架构师特别关注后五个**（B+ Tree、Bloom Filter、LSM、一致性哈希、HLL）—— 它们是你做存储选型、缓存设计、分片决策时绕不开的。

### 2.2 算法必备类别

- **复杂度分析**：能在 30 秒内说出一段代码的 Big-O，及空间复杂度。
- **排序**：归并/快排/堆排各自最佳场景；理解外部排序（大数据不能进内存怎么办）。
- **图算法**：BFS/DFS、最短路径（Dijkstra）、拓扑排序（依赖解析）。
- **动态规划**：理解状态、转移、子问题重叠。架构里用得不多，但训练拆问题的能力。
- **二分**：经典写法以外，**在答案空间上二分**（最小化最大值这种）是高阶武器。

### 2.3 架构师视角

工程里很少手写算法，但**选错数据结构 = 性能差 100 倍** 的事每天都在发生。

例：
- 用 `List.contains()` 在 10 万元素里查找 → 改成 `HashSet` → 性能从 O(n) → O(1)
- 用数据库 LIKE 做模糊匹配 → 改成倒排索引（Elasticsearch）→ 从 5 秒 → 50ms
- 用 SQL `COUNT(DISTINCT)` 算 UV → 改成 HyperLogLog → 从扫表 → O(1) 估算

## 3. 操作系统

### 3.1 进程与线程

- **进程**：独立地址空间、独立资源、切换重。
- **线程**：共享地址空间、轻量切换，但需要同步原语。
- **协程 / 用户态线程**：用户态调度，无内核切换开销（Go goroutine、Java Virtual Thread）。

架构含义：高并发选模型时，**每连接一线程**适合连接少长时活动；**事件循环 + 协程**适合海量短连接（Nginx、Redis、Node.js 都这条路）。

### 3.2 内存

- 虚拟内存：每个进程独立虚拟地址空间，按需 page 进物理内存。
- 缺页中断：访问没在物理内存的页 → 陷入内核 → 从磁盘读 → 慢 1000 倍。
- **OOM Killer**：Linux 内存吃光时随机杀进程。容器化部署必须设 limit 并预留 buffer。
- 内存映射 (mmap)：把文件映射到地址空间，零拷贝读，被 Kafka / RocketMQ 大量使用。

### 3.3 IO 模型

| 模型 | 特点 | 典型代表 |
|---|---|---|
| 阻塞 IO | 一个线程一个连接 | 传统 Java BIO |
| 非阻塞 IO + 轮询 | 浪费 CPU | 几乎不用 |
| IO 多路复用 (select/poll/epoll/kqueue) | 一个线程管 N 个连接 | Nginx、Redis |
| 异步 IO (AIO, io_uring) | 内核完成后回调 | 新一代高性能存储 |

架构含义：**epoll 是现代高并发服务器的基石**。理解它的边沿/水平触发、惊群问题，能让你看懂 Netty、Nginx、Redis 的核心设计。

### 3.4 文件系统

- 顺序写 >>> 随机写（磁盘和 SSD 都如此，只是 SSD 差距小）
- **append-only 日志**是高吞吐存储的核心套路（Kafka、HDFS、所有 LSM 类存储）
- fsync 决定可靠性：不 fsync 数据可能丢，频繁 fsync 性能崩

### 3.5 必懂的内核工具

- `top` / `htop` / `vmstat` / `iostat` / `pidstat` —— 看 CPU、内存、IO 谁吃了资源
- `strace` —— 看进程在调哪些 syscall
- `tcpdump` / `wireshark` —— 抓包
- `perf` / `flamegraph` —— 找 CPU 热点
- `lsof` —— 看进程打开了什么文件/连接

架构师面对线上诡异问题时，能否在 30 分钟内用这些工具定位到根因，是分水岭能力。

## 4. 计算机网络

### 4.1 TCP/IP 关键点

- **三次握手 / 四次挥手**：理解 TIME_WAIT 为什么存在、CLOSE_WAIT 堆积说明什么。
- **滑动窗口、拥塞控制**：解释为什么"带宽够但传输慢"。
- **MSS / MTU**：包被切片的临界值，跨网络环境时容易踩。
- **Nagle 算法 / 延迟 ACK**：理解为什么小包发送有时会卡。
- **Keep-Alive**：长连接的成本与价值。

### 4.2 HTTP 演进

| 版本 | 关键特性 | 架构含义 |
|---|---|---|
| HTTP/1.0 | 一请求一连接 | 慢 |
| HTTP/1.1 | Keep-Alive、Pipelining | 仍受队头阻塞限制 |
| HTTP/2 | 多路复用、Header 压缩、Server Push | 一个 TCP 连接传所有，但 TCP 层仍有队头阻塞 |
| HTTP/3 (QUIC) | 基于 UDP、连接迁移、0-RTT | 移动端、跨大陆显著提升 |

### 4.3 TLS / HTTPS

- 握手过程：证书校验、密钥协商、对称密钥建立。
- TLS 1.3 把握手从 2-RTT 砍到 1-RTT，0-RTT 可选。
- **架构含义**：内网调用是否走 TLS，是性能 vs 安全的经典 trade-off。零信任架构下倾向于全 TLS（mTLS）。

### 4.4 DNS

- 解析过程：本地 → 递归 DNS → 根 → 顶级域 → 权威 DNS。
- DNS 缓存层级多，TTL 决定故障切换速度。
- **常见坑**：JVM 默认 DNS 缓存 forever；切换 IP 后服务不通。

### 4.5 网络层架构概念

- **CDN**：把静态/动态内容推到离用户近的边缘。
- **反向代理 / 正向代理**：方向不同，作用不同。
- **VIP / LVS / 四层负载均衡 / 七层负载均衡**：分清楚谁在哪一层。
- **NAT / 内外网穿透**：理解云上跨 VPC、跨账号的连通方式。

## 5. 数据库基础

### 5.1 关系型数据库核心

- **B+ 树索引**：聚簇索引 vs 二级索引，回表是什么。
- **事务与 ACID**：原子、一致、隔离、持久。
- **隔离级别**：读未提交、读已提交、可重复读、串行化 —— 各自允许哪些异常（脏读、不可重复读、幻读）。
- **MVCC**：MySQL InnoDB / PostgreSQL 都用，理解 undo log 和 read view。
- **锁**：行锁、间隙锁、意向锁、表锁；死锁是怎么形成的。

### 5.2 SQL 必须熟

- JOIN 的三种实现：nested loop / hash join / merge join，谁快取决于数据量和索引。
- 执行计划阅读：`EXPLAIN` 能看出来索引用没用、扫了多少行、有没有临时表。
- 慢 SQL 套路：少用 SELECT *、避免函数包裹索引列、注意隐式类型转换。

### 5.3 缓存与存储引擎概念

- **写优化 vs 读优化**：LSM Tree 写快读慢，B+ Tree 读快写慢。
- **WAL (Write-Ahead Log)**：所有可靠存储的基础。
- **Buffer Pool**：脏页、checkpoint、刷盘。

### 5.4 架构师视角

不是要会运维 MySQL，而是要能：
- 看一个表设计，估算 QPS 上限。
- 听到"慢"，能问到底是 IO 慢、锁等待还是 CPU 飙。
- 选型时知道 PG/MySQL/MongoDB/Cassandra 的"擅长的形状"。

详细的存储选型在 [06-data-architecture.md](06-data-architecture.md)。

## 6. 反模式

| 反模式 | 为什么坏 |
|---|---|
| 只会一门语言但自称"语言无关" | 没有比较就没有取舍直觉 |
| 算法只为面试刷 | 真到选数据结构时仍然抓瞎 |
| 不懂 epoll 就讨论高并发架构 | 等于不懂引擎讨论赛车 |
| 用 ORM 就不学 SQL | ORM 屏蔽不了执行计划问题 |
| 不会用 perf/strace/tcpdump | 线上问题永远靠日志猜 |
