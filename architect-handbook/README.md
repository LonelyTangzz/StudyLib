# 架构师能力手册

> 一本写给"准备/正在/已经"成为软件架构师的人的速查与精读手册。覆盖从技术基础到业务建模、从分布式系统到软技能的完整知识地图。
>
> **目标**：你读完后能回答两件事 —— "做架构师要会什么"、"每个能力具体怎么练"。

---

## 0. 怎么用这本手册

- **顺序读**：1 → 16，从心智到成长路径线性递进，适合系统补课。
- **跳读**：按需查阅某章，每章都自包含，独立可读。
- **当成 checklist 用**：每章末尾都有"自检清单"，对照自己当前能力打分。

每章统一结构：

```
1. 这章在讲什么（一段话）
2. 核心概念（必须懂的术语/原理）
3. 模式与实践（怎么用）
4. 反模式与坑（别这么干）
5. 自检清单（能不能讲清/能不能做到）
```

## 3. 用 3 个常见问题校准认知

**Q1: 架构师就是写文档不写代码？**
错。**资深架构师必须能读懂、必要时能改任何关键路径上的代码**。文档是产物，代码是真相。脱离代码的架构师会被工程师无声忽视。

**Q2: 架构师的核心交付物是什么？**
不是 PPT，不是 UML 图。是**被多团队接受并执行的技术决策**。决策的载体可以是 ADR、技术方案、原型、参考实现，但本质是"对齐 + 落地"。

**Q3: 微服务/中台/云原生是不是架构师的必备？**
是工具箱里的东西。但**首要能力是"知道什么场景该用什么、不该用什么"**。盲目上微服务、不分场景搞中台的架构师，是公司最贵的负债。
